HTML Tutorial

Basic Structure

Every HTML document needs a basic structure. Here's the foundation: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Your Page Title</title>
</head>
<body>
    <!-- Your content goes here -->
</body>
</html>

CSS Tutorial

Selectors and Properties

CSS uses selectors to style HTML elements:

/* Element Selector */
p {
    color: #333;
    font-size: 16px;
    margin-bottom: 1rem;
}

/* Class Selector */
.button {
    background-color: #2563eb;
    color: white;
    padding: 0.5rem 1rem;
    border-radius: 4px;
}

/* ID Selector */
#header {
    background: #f8fafc;
    padding: 1rem;
}

Flexbox Layout

Flexbox makes it easy to create flexible layouts:

.container {
    display: flex;
    justify-content: space-between;
    align-items: center;
    gap: 1rem;
}

.item {
    flex: 1;
    padding: 1rem;
}

JavaScript Tutorial

Variables and Data Types

JavaScript has several ways to declare variables:

// Variable declarations
let name = "John";  // String
const age = 25;     // Number
var isActive = true;  // Boolean

// Array
const fruits = ["apple", "banana", "orange"];

// Object
const person = {
    name: "John",
    age: 25,
    isStudent: true
};

Functions

Functions are blocks of reusable code:

// Function declaration
function greet(name) {
    return Hello, ${name}!;
}

// Arrow function
const multiply = (a, b) => a * b;

// Function with default parameter
function setColor(color = "blue") {
    console.log(Color set to ${color});
}

Web APIs Tutorial

Fetch API

The Fetch API provides a way to make network requests and handle responses. It's a modern alternative to XMLHttpRequest, offering a cleaner and more powerful way to interact with APIs:

// Example: Fetching data from an API
fetch('https://jsonplaceholder.typicode.com/posts/1')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

Note: This code fetches data from a sample API and logs it to the console.

Geolocation API

The Geolocation API allows you to access the user's current location (with their permission). It's commonly used for location-based services:

// Example: Getting user's current location
navigator.geolocation.getCurrentPosition(
  (position) => {
    const { latitude, longitude } = position.coords;
    console.log(`Latitude: ${latitude}, Longitude: ${longitude}`);
  },
  (error) => console.error('Error:', error.message)
);

Note: This code retrieves the user's latitude and longitude.

Local Storage API

The Local Storage API allows you to store data in the browser. The data is stored as key-value pairs and persists even after the browser is closed:

// Example: Using Local Storage
localStorage.setItem('username', 'JohnDoe'); // Save data
const user = localStorage.getItem('username'); // Retrieve data
console.log(user); // Output: JohnDoe
localStorage.removeItem('username'); // Remove data

Note: This code shows how to save, retrieve, and delete data from local storage.

ARIA Roles and Screen Reader Support

Introduction to ARIA

ARIA (Accessible Rich Internet Applications) roles and attributes are used to make web content more accessible to users with disabilities. These attributes help screen readers and other assistive technologies better understand the structure and functionality of your web pages.

Common ARIA Roles

Here are some commonly used ARIA roles to improve accessibility:

<!-- Landmark roles -->
<header role="banner">Site Header</header>
<nav role="navigation">Main Menu</nav>
<main role="main">Main Content</main>
<footer role="contentinfo">Footer Information</footer>

<!-- Button role -->
<div role="button" tabindex="0">Clickable Button</div>

<!-- Alert role -->
<div role="alert">This is an important message!</div>

Using ARIA Attributes

ARIA attributes provide additional information about elements, enhancing their accessibility:

<!-- ARIA Label -->
<button aria-label="Close Menu">×</button>

<!-- ARIA Hidden -->
<div aria-hidden="true">Hidden from screen readers</div>

<!-- ARIA Live -->
<div aria-live="polite">This content will be updated dynamically</div>

<!-- ARIA Expanded -->
<button aria-expanded="false">Show Details</button>

Example: Accessible Navigation Menu

Here's an example of a fully accessible navigation menu with ARIA roles and attributes:

<nav role="navigation" aria-label="Primary Navigation">
    <ul>
        <li><a href="index.html" aria-current="page">Home</a></li>
        <li><a href="about.html">About Us</a></li>
        <li><a href="services.html">Services</a></li>
        <li><a href="contact.html">Contact</a></li>
    </ul>
</nav>

Note: The aria-current attribute is used to indicate the currently active page.

Using ARIA for Screen Readers

ARIA attributes can greatly improve the experience for screen reader users:

<!-- Descriptive label for screen readers -->
<input type="text" aria-label="Search the site" placeholder="Search...">

<!-- Grouping form elements -->
<fieldset>
    <legend>Contact Information</legend>
    <label for="name">Name:</label>
    <input type="text" id="name">
    <label for="email">Email:</label>
    <input type="email" id="email">
</fieldset>

Conclusion

Using ARIA roles and attributes is essential for building inclusive and accessible websites. By implementing these practices, you ensure a better user experience for everyone, including those relying on assistive technologies.

Responsive Design Tutorial

Introduction to Responsive Design

Responsive Design is a web design approach that makes web pages render well on a variety of devices and window or screen sizes. It uses CSS media queries, flexible grid layouts, and scalable images to adapt the layout to the user's device.

Using Media Queries

Media queries allow you to apply different styles based on the device's screen size or other characteristics:

@media screen and (max-width: 768px) {
    body {
        background-color: lightblue;
    }

    .container {
        flex-direction: column;
    }
}

Flexbox and Grid for Layouts

Flexbox and Grid are layout models that make it easier to design responsive layouts. Flexbox is ideal for linear layouts, while Grid is better for two-dimensional layouts:

.container {
    display: flex;
    flex-wrap: wrap;
    justify-content: space-between;
}

.item {
    flex: 1 1 200px;
    padding: 1rem;
}

@media screen and (max-width: 600px) {
    .container {
        flex-direction: column;
    }
}

Responsive Images

To make images responsive, you can use the `max-width: 100%` rule and `height: auto` to ensure images scale according to their container:

img {
    max-width: 100%;
    height: auto;
}

Mobile First Design

Mobile-first design focuses on designing the mobile version of the website first and then scaling up for larger devices. Use the following CSS rule to implement it:

@media screen and (min-width: 768px) {
    body {
        font-size: 18px;
    }
}

Example: Responsive Navigation

This is an example of a simple responsive navigation menu using media queries:

<nav>
    <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">About</a></li>
        <li><a href="#">Services</a></li>
        <li><a href="#">Contact</a></li>
    </ul>
</nav>

<style>
nav ul {
    display: flex;
    justify-content: space-around;
}

@media screen and (max-width: 768px) {
    nav ul {
        flex-direction: column;
        align-items: center;
    }
}
</style>

Conclusion

Responsive design ensures that your website provides an optimal experience across a wide range of devices. Using media queries, flexible layouts, and scalable images, you can create sites that adapt to the user's screen size and provide a seamless user experience.
